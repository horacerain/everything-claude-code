## 二、需要你手动触发的功能（Commands）

### 🔴 高频使用（推荐优先记住）

| 命令 | 用法 | 什么情况下用 |
|------|------|-------------|
| `/plan 任务描述` | 规划功能实现 | 开始新功能前 |
| `/code-review` | 代码审查 | 写完代码后 |
| `/build-fix` | 修复构建错误 | 构建失败时 |

| 命令 | 用法 | 什么情况下用 |
|------|------|-------------|

| `/e2e 描述` | 端到端测试 | 测试关键用户流程 |

| `/test-coverage` | 分析覆盖率 | 查看测试覆盖情况 |

## 三、使用示例（实际对话场景）

### 场景 1：开始一个新功能

```
你: /plan "添加用户登录功能，包括用户名密码和验证码"
     ↓
OpenCode: 
1. 先问你几个问题确认需求
2. 列出详细的实现步骤
3. 列出可能的风险
4. 等你确认后再开始写代码
```

### 场景 2：写完代码后

```
你: /code-review
     ↓
OpenCode: 
1. 检查你刚写的代码
2. 列出发现的问题（安全性、代码质量等）
3. 给出修复建议
```

### 场景 3：构建失败了

```
你: /build-fix
     ↓
OpenCode: 
1. 运行类型检查
2. 找出所有错误
3. 一个一个修复
4. 验证修复后构建成功
```

### 场景 4：开发新功能（推荐用 TDD）

```
你: /tdd "实现一个购物车功能"
     ↓
OpenCode: 
1. 先写测试用例
2. 再写代码
3. 确保测试通过
4. 要求 80%+ 测试覆盖率
```

### 场景 5：处理敏感代码

```
你: /security "检查用户注册和登录的代码"
     ↓
OpenCode: 
1. 检查是否有安全漏洞
2. 检查密码是否加密
3. 检查是否有 SQL 注入风险
4. 给出安全建议
```

## 四、Agents 是什么？怎么用？

### 概念解释

**Agent = 专业代理** = 一个训练好的 AI 角色，专门做一件事。

比如 `code-reviewer` 就是一个专门做代码审查的 AI，你不需要告诉它怎么审查，它知道。

### 你需要直接使用 Agents 吗？

**一般不需要**。Commands 会自动调用对应的 Agent。

```
你输入 /code-review
    ↓
OpenCode 调用 code-reviewer 这个 Agent
    ↓
Agent 执行代码审查
```

### 什么情况下可能需要知道 Agents？

当你看到配置文件中提到某个 Agent 时，知道它的用途：
- `planner` → 规划师
- `code-reviewer` → 代码审查员
- `security-reviewer` → 安全审计员
- `tdd-guide` → TDD 教练

## 五、完全自动的部分（你不需要做任何事）

### Hooks（自动触发）

| Hook | 什么时候触发 | 做什么 |
|------|-------------|-------|
| 自动格式化 | 保存 .ts/.js 文件 | Prettier 格式化代码 |
| 类型检查 | 执行命令后 | 检查 TypeScript 错误 |
| console.log 警告 | 编辑文件时 | 提醒删除调试代码 |
| 安全检查 | 执行命令前 | 检查是否有密钥泄露 |

### Skills（AI 自动使用）

当 AI 处理特定任务时，会自动参考这些技能：
- `tdd-workflow` → TDD 开发流程
- `security-review` → 安全审查清单
- `coding-standards` → 编码规范
- `e2e-testing` → E2E 测试最佳实践
- 等 11 个...

## 七、常见问题

**Q: 我不知道该用哪个命令怎么办？**
A: 直接问 OpenCode："我应该用哪个命令来处理这个任务？"

**Q: 输入命令后会发生什么？**
A: OpenCode 会切换到对应的专业代理模式，帮你完成任务，完成后自动回到正常对话。

**Q: 命令执行一半可以中断吗？**
A: 可以，直接告诉 OpenCode "停止"或"取消"即可。

## 八、卸载方法

如果不再需要 ECC 插件，按以下步骤卸载：

```bash
# 1. 删除插件目录
rm -rf ~/.config/opencode/plugins/ecc/

# 2. 编辑配置文件，删除 ECC 配置区块
#    （配置文件中包含 "===== ECC 开始 =====" 到 "===== ECC 结束 =====" 的注释区块）
nano ~/.config/opencode/opencode.json
```
